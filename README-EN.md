## 💡 Why Simon Refactor Skill?

When dealing with legacy systems spanning millions of lines, AI-assisted refactoring often fails due to:

  - **Context Overload**: AI reads too many irrelevant files, leading to "hallucinations" or lost context.
  - **Fear of Breaking Things**: Lack of safety nets makes refactoring high-risk for production.
  - **Lack of Structure**: Random edits without a systematic governance plan.

**Simon-Refactor-Skill** solves this by breaking down complex refactoring into **4 phases and 12 standardized steps**, providing ready-to-use Claude Code Prompts and Sub-agent configurations.

-----

## 🌟 Key Features

  - 🛠 **Interactive Task Dashboard**: Track your refactoring progress in real-time with an integrated checklist.
  - 🤖 **Optimized Prompts**: Battle-tested prompts for reducing cyclomatic complexity, eliminating duplicate code, and modernizing syntax.
  - 📦 **Zero Dependency**: A portable, single-file HTML web app. No installation required—just open `index.html`.
  - 🛡 **Safety-First Approach**: Built-in strategies for Characterization Testing and Git safety baselines.
  - ⚡ **Parallel Execution**: Expert guidance on using Git Worktrees to scale refactoring across multiple Claude instances.

## 🗺 Refactoring Methodology

1.  **Preparation**: Initialize `CLAUDE.md` to establish the "Project Brain."
2.  **Exploration**: Full-scan analysis to identify high-risk technical debt.
3.  **Safety Net**: Generate Characterization Tests to build an automated CI guardrail.
4.  **Batch Refactoring**: Manage context windows and execute modular refactoring.
5.  **Validation & Governance**: Integration checks and long-term quality inspection loops.
